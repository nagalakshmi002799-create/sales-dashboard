Sales Dashboard Documentation
============================

Overview
--------
This document describes the sales dashboard project, a comprehensive React-based application for monitoring user growth, city filters, and customer performance. The project features a collapsible sidebar, responsive mobile drawer menu, responsive design, and real-time data visualization.

Tech Stack
----------
- **Frontend Framework**: React 18.3.1 with Hooks
- **Build Tool**: Vite 8.0.12
- **State Management**: Redux Toolkit 1.9.5
- **Styling**: Bootstrap 5.3.3 + Custom CSS
- **Data Fetching**: Native Fetch API
- **Package Manager**: npm
- **Development**: ESLint for code quality

Dashboard Layout
----------------
**Desktop (>991px)**:
- Two-column layout: Sidebar (280px fixed, collapsible to 96px) + Main content
- Sidebar navigates to: Overview, Customers, Analytics sections
- Main content: Topbar → Summary Cards → Customer Section → Overview Charts → Analytics

**Mobile/Tablet (≤991px)**:
- Single-column layout: Fixed sidebar drawer + Full-width content
- Sidebar slides in from left as a drawer menu
- Hamburger button (☰) in topbar to toggle sidebar drawer
- Auto-closes sidebar after navigation on mobile

Low Level Design
----------------

### Architecture Overview
The application follows a component-based architecture with clear separation of concerns:

1. **Presentation Layer**: React components handle UI rendering and user interactions
2. **State Management Layer**: Redux Toolkit manages global application state
3. **Data Layer**: Async thunks handle API calls and data fetching
4. **Styling Layer**: Bootstrap + custom CSS for responsive design
5. **Mobile Optimization**: Fixed drawer sidebar, hamburger menu, responsive layout

### Component Hierarchy
```
App (Root Component)
├── Sidebar (Navigation - Fixed/Drawer)
├── Topbar (Header with Hamburger on Mobile)
├── SummaryCards (Colorful KPI Cards)
└── Main Content
    ├── FilterPanel (Search/Filter)
    ├── UserTableSection (Data Table)
    │   └── UserTable (Table Rows)
    ├── DashboardMain (Charts & Insights)
    │   ├── Total Revenue Chart
    │   ├── Visitor Insights
    │   ├── Customer Satisfaction
    │   ├── Target vs Reality
    │   └── Top Products
    └── DashboardCharts (City Distribution Analytics)
```

### Data Flow Architecture
```
User Interaction → Component → Action Dispatch → Reducer → State Update → Component Re-render

1. User clicks refresh → dispatch(fetchUsers()) → usersSlice reducer → loading: true
2. API call completes → fetchUsers.fulfilled → users: data, loading: false
3. Components re-render with new data → UI updates
4. User filters/searches → dispatch(setSearchText()) → selectFilteredUsers computes filtered data
```

### State Management Design
- **Centralized State**: All application state managed in Redux store
- **Slice-based Architecture**: usersSlice handles user-related state and filters
- **Async Operations**: createAsyncThunk for API calls
- **Selectors**: createSelector for computed state (filtering, searching)
- **Immutability**: Redux Toolkit handles state updates immutably

State Structure:
```javascript
{
  users: [],           // Array of user objects
  loading: false,      // Loading state for async operations
  error: null,         // Error messages
  searchText: '',      // Search filter text
  selectedCity: 'All', // City filter dropdown
  sortDirection: 'asc' // Sort direction toggle
}
```

Reusable Components
-------------------

### 1. SummaryCards Component
**Purpose**: Display KPI metrics in colorful themed cards
**Props**: None - Displays: Total Sales, Total Orders, Products Sold, New Customers
**Features**:
- Multiple colors: Blue, Yellow, Green, Purple backgrounds
- Responsive grid: 1 column mobile, 2 columns tablet, 4 columns desktop
- Trend indicators with colored badges
- Visual hierarchy with distinct colors
**Styling**: Each card has unique background color and text color combinations

### 2. FilterPanel Component
**Purpose**: Search and filter controls for data tables
**Props**:
- `searchText`, `onSearchChange` - Search by name or email
- `selectedCity`, `onCityChange` - Filter by city dropdown
- `sortDirection`, `onSortChange` - Toggle ascending/descending sort
- `cityOptions` - Array of available cities
**Features**:
- Real-time search filter
- City multi-filter selection
- Sort direction toggle
- Responsive layout

### 3. UserTable Component
**Purpose**: Display user data in table format
**Props**: 
- `{ loading, error, users }` - User array with metadata
**Features**:
- Responsive table with scrolling on mobile
- Loading and error states
- User information: Name, Email, Company, City, Phone
- Empty state handling

### 4. Sidebar Component
**Purpose**: Navigation sidebar (Fixed on desktop, Drawer on mobile)
**Props**:
- `activeSection` - Current active navigation item
- `collapsed` - Boolean for collapse state (desktop only)
- `onNavigate` - Navigation callback
- `onToggle` - Collapse/drawer toggle callback
**Features**:
- Desktop: Collapsible sidebar (280px ↔ 96px)
- Mobile: Drawer menu that slides from left
- Navigation items: Overview, Customers, Analytics
- Support button with conditional text
- Auto-closes after navigation on mobile

### 5. Topbar Component
**Purpose**: Application header with actions and mobile menu toggle
**Props**: 
- `{ onRefresh, onToggleSidebar }` - Callbacks
**Features**:
- Responsive layout (stacks on mobile)
- Hamburger menu button (☰) visible only on mobile
- Refresh data button
- Profile display with admin info
- Dashboard title display

### 6. DashboardMain Component
**Purpose**: Display main charts and insights
**Features**:
- Total Revenue chart (offline vs online)
- Visitor Insights line chart
- Customer Satisfaction metrics
- Target vs Reality comparison
- Top Products bar chart

### 7. DashboardCharts Component
**Purpose**: Analytics overview with city distribution
**Features**:
- City distribution bar chart
- Top 5 cities by active users
- Progress bars with percentages

Component Design Patterns
-------------------------
1. **Functional Components**: All components use modern React hooks
2. **Props Interface**: Clear prop contracts for each component
3. **CSS Classes**: Bootstrap + custom CSS for styling
4. **Responsive Classes**: Bootstrap grid system (col-12, col-sm-*, col-lg-*)
5. **Accessibility**: ARIA labels, keyboard navigation, semantic HTML
6. **Loading States**: Consistent loading UI with spinners
7. **Error States**: Graceful error messages for failures

API Integration
---------------
- **Data Source**: JSONPlaceholder API (https://jsonplaceholder.typicode.com/users)
- **HTTP Method**: GET request for user data
- **Error Handling**: Network errors and API failures with user feedback
- **Loading States**: UI feedback (spinners) during data fetching
- **Data Transformation**: Raw API data transformed for component consumption
- **Auto-refresh**: Refresh button in topbar to reload data

Build and Deployment
--------------------
- **Development**: `npm run dev` - Hot reload development server on port 5173
- **Production Build**: `npm run build` - Optimized production bundle
- **Preview**: `npm run preview` - Local production preview
- **Linting**: `npm run lint` - ESLint code quality checks
- **Bundle Analysis**: Vite provides detailed bundle size reports

Performance Optimizations
-------------------------
1. **Code Splitting**: Vite handles automatic code splitting
2. **Tree Shaking**: Unused code removed in production builds
3. **Lazy Loading**: Components loaded on demand
4. **Memoization**: React.memo for expensive re-renders
5. **Efficient Selectors**: Redux selectors prevent unnecessary re-renders
6. **Minimal Bundle**: Bootstrap utilities tree-shaken for size reduction
7. **CSS Optimization**: Unused CSS removed in production
