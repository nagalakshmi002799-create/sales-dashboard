Sales Dashboard Documentation
============================

Overview
--------
This document describes the sales dashboard project, a comprehensive React-based application for monitoring user growth, city filters, and customer performance. The project features a collapsible sidebar, responsive design, and real-time data visualization.

Tech Stack
----------
- **Frontend Framework**: React 18.3.1 with Hooks
- **Build Tool**: Vite 8.0.12
- **State Management**: Redux Toolkit 1.9.5
- **Styling**: Bootstrap 5.3.3 + Custom CSS
- **Data Fetching**: Native Fetch API
- **Package Manager**: npm
- **Development**: ESLint for code quality

Low Level Design
----------------

### Architecture Overview
The application follows a component-based architecture with clear separation of concerns:

1. **Presentation Layer**: React components handle UI rendering and user interactions
2. **State Management Layer**: Redux Toolkit manages global application state
3. **Data Layer**: Async thunks handle API calls and data fetching
4. **Styling Layer**: Bootstrap + custom CSS for responsive design

### Component Hierarchy
```
App (Root Component)
├── Sidebar (Navigation)
├── Topbar (Header)
└── Main Content
    ├── DashboardHeader (Page Title)
    ├── SummaryCards (KPI Cards)
    ├── FilterPanel (Search/Filter)
    ├── UserTableSection (Data Table)
    │   └── UserTable (Table Rows)
    ├── DashboardCharts (Visualizations)
    └── PerformancePanel (Metrics)
```

### Data Flow Architecture
```
User Interaction → Component → Action Dispatch → Reducer → State Update → Component Re-render

1. User clicks refresh → dispatch(fetchUsers()) → usersSlice reducer → loading: true
2. API call completes → fetchUsers.fulfilled → users: data, loading: false
3. Components re-render with new data → UI updates
```

### State Management Design
- **Centralized State**: All application state managed in Redux store
- **Slice-based Architecture**: usersSlice handles user-related state
- **Async Operations**: createAsyncThunk for API calls
- **Selectors**: createSelector for computed state
- **Immutability**: Redux Toolkit handles state updates immutably

State Structure:
```javascript
{
  users: [],           // Array of user objects
  loading: false,      // Loading state for async operations
  error: null,         // Error messages
  searchText: '',      // Search filter
  selectedCity: 'All', // City filter
  sortDirection: 'asc' // Sort direction
}
```

### Responsive Design Architecture
- **Mobile-First**: Base styles for mobile, enhanced for larger screens
- **Breakpoint Strategy**: 991px (tablet/mobile), 1199px (desktop)
- **Flexbox Layout**: Main layout uses flexbox for flexibility
- **Conditional Rendering**: Sidebar behavior changes based on screen size
- **Progressive Enhancement**: Features enabled based on screen capabilities

Approach
--------
1. Maintain a clear two-column dashboard layout on desktop.
2. Introduce a collapsible sidebar to increase workspace area and improve user focus.
3. Add a fixed back-to-top button that appears when the user scrolls down.
4. Use responsive media queries to adapt the layout for smaller viewports.
5. Keep the implementation simple, accessible, and easy to maintain.

Components
----------
- `src/components/Sidebar.jsx`
  - Contains the navigation panel with logo-based collapse/expand.
  - The "AD" logo button triggers sidebar collapse/expand on desktop (>991px).
  - Supports `collapsed` mode, which shrinks the sidebar to 96px and hides text labels.
  - Displays icon placeholders for each nav item using their first letter.
  - Support button shows as "?" in expanded mode and "S" in collapsed mode.
  - On mobile (<991px), the sidebar is always full-width and expanded for better usability.

- `src/App.jsx`
  - Hosts the main dashboard layout and application shell.
  - Uses React state to manage:
    - `sidebarCollapsed` for the sidebar toggling.
    - `showBackToTop` for the back-to-top button visibility.
    - `activeSection` for tracking the highlighted navigation item.
  - Uses section refs to scroll smoothly to `overview`, `customers`, `analytics`, and `reports`.
  - Adds a fixed `Back to top` button when the page scroll position passes a threshold.

- `src/components/Topbar.jsx`
  - Displays dashboard header text, refresh action, and profile summary.
  - Uses responsive flex layout so actions wrap cleanly on narrow screens.

Functionality
-------------
- Sidebar collapse and expand:
  - Click the "AD" logo button to toggle `sidebarCollapsed` state on desktop (>991px).
  - When collapsed, sidebar shrinks from 280px to 96px smoothly.
  - Text labels and branding info hide; icons remain visible.
  - Support button displays "S" icon when collapsed, full text when expanded.
  - On mobile (<991px), collapse state is disabled and sidebar is always full-width.

- Support button styling:
  - Expanded: shows full "Support" text with "?" icon
  - Collapsed (desktop only): shows compact "S" icon matching nav items
  - Hover state: slight background color change for visual feedback

- Section navigation:
  - Each sidebar item scrolls the matching dashboard section into view with `scrollIntoView({ behavior: 'smooth' })`.

- Back-to-top feature:
  - A `window` scroll listener updates `showBackToTop`.
  - The button appears after the user scrolls down and uses `window.scrollTo({ top: 0, behavior: 'smooth' })`.

- Responsive behavior:
  - Desktop (>991px): Two-column grid with collapsible sidebar (collapse only at this breakpoint).
  - Tablet/Mobile (<991px): Single-column layout with full-width, always-expanded sidebar.
  - Content area automatically expands when sidebar collapses on desktop.
  - All interactive elements adapt sizing and spacing for smaller screens.

CSS and Styling
---------------
- `src/App.css`
  - `.app-shell`
    - Uses Flexbox layout: `display: flex` with `flex-direction: column` on mobile
    - Sidebar has `flex-shrink: 0` to maintain width, main content has `flex: 1`
    - Responsive gap and padding for different screen sizes

  - `.sidebar`
    - Fixed width sidebar (280px desktop, responsive scaling)
    - Gradient background (dark blue) and rounded shape for visual hierarchy
    - Smooth transitions for width changes during collapse
    - Min-height ensures proper proportions on all screen sizes

  - `.sidebar-logo-btn`
    - Clickable button styled with purple background
    - Hover effect: color change and slight scale increase
    - Triggers collapse/expand toggling on desktop only

  - `.sidebar-collapsed` (desktop only)
    - Shrinks sidebar to compact width (96px)
    - Hides text labels and branding info
    - Centers icon-only nav buttons

  - `.support-btn` and `.support-icon`
    - Support button has semi-transparent background with light border
    - Icon displays "?" (expanded) or "S" (collapsed)
    - On mobile, always shows full text "Support"

  - `.nav-icon` and `.nav-text`
    - Icons provide visual anchors for nav items
    - Text hides in collapsed mode on desktop
    - Always visible on mobile

  - `.back-to-top`
    - Fixed position button (bottom-right)
    - Smooth scroll to top animation
    - Responsive positioning for different screen sizes

  - Media queries
    - `@media (max-width: 1199px)`: sidebar reduces to 240px for narrower desktops
    - `@media (max-width: 1075px)`: sidebar reduces to 220px
    - `@media (max-width: 1005px)`: sidebar reduces to 200px
    - `@media (max-width: 991px)`: switches to single-column, disables collapse feature
    - `@media (max-width: 767px)`: adjusts padding, radius, and button placement
